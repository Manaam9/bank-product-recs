# 🏦 Рекомендательная система банковских продуктов

## 📌 Описание проекта

Цель проекта — разработать рекомендательную систему, которая предсказывает, какие банковские продукты (кредиты, депозиты, инвестиционные продукты и др.) стоит предложить клиенту.

Проект реализует полный ML-пайплайн:

* анализ данных (EDA)
* генерация признаков
* обучение моделей
* организация экспериментов (MLflow)
* деплой модели (FastAPI + Docker)
* мониторинг и сопровождение

Данные содержат историю клиентов банка за 1.5 года с ежемесячными обновлениями продуктового портфеля.

---

## 🎯 Постановка задачи

### Бизнес-задача

Повысить конверсию в подключение банковских продуктов за счёт персонализированных рекомендаций.

### ML-задача

Для каждого клиента предсказать набор новых продуктов, которые он с высокой вероятностью подключит в следующем месяце.

Задача формулируется как **ranking / multi-label classification**.

---

## 📊 Метрики

### ML-метрики

* Precision@K
* Recall@K
* MAP@K

Выбор обусловлен тем, что пользователю показывается ограниченное число рекомендаций.

### Бизнес-метрики

* Conversion Rate
* Uptake Rate
* ARPU (средний доход на пользователя)

### Технические метрики

* latency API
* количество ошибок
* throughput

## 🔍 EDA (исследование данных)

В ходе анализа данных выявлены следующие закономерности:

* значительная часть клиентов имеет ограниченный набор продуктов
* распределение продуктов сильно несбалансировано
* активность клиента влияет на вероятность подключения новых продуктов
* наблюдается временная динамика подключений

EDA выполнен в ноутбуке: `notebooks/eda.ipynb`

## 🧬 Генерация признаков

Использовались следующие группы признаков:

### Клиентские

* возраст, пол, доход
* сегмент клиента
* стаж

### Поведенческие

* активность клиента
* история использования продуктов

### Исторические

* наличие продуктов в предыдущем месяце
* количество продуктов

### Временные

* месяц
* длительность взаимодействия с банком
