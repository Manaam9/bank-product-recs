# Рекомендательная система банковских продуктов

## Описание проекта

Цель проекта — разработать рекомендательную систему, которая предсказывает, какой банковский продукт предложить клиенту.

Проект реализуется в формате production-пайплайна:

* анализ данных
* постановка ML-задачи
* организация экспериментов
* моделирование
* деплой и мониторинг

---

## Цель проекта

Построить модель, которая предсказывает:

> какие новые банковские продукты клиент с наибольшей вероятностью оформит в следующем месяце

---

## Постановка задачи

Задача формулируется как:

> предсказание новых продуктов клиента на основе его предыдущего состояния и истории

Особенности:

* временная структура данных
* multi-label постановка
* сильный дисбаланс
* задача recommendation / ranking

---

## Описание данных

Используются данные о клиентах банка за **1.5 года**.

### Основные характеристики:

* ежемесячные срезы (`fecha_dato`)
* клиентская информация
* поведенческие признаки
* продуктовый портфель

Каждая строка:

> клиент + месяц + набор продуктов

---

### Типы признаков

* идентификатор: `ncodpers`
* временные: `fecha_dato`, `fecha_alta`
* демография: `sexo`, `age`, `pais_residencia`
* поведение: `ind_actividad_cliente`, `canal_entrada`
* финансы: `renta`
* продуктовые признаки: `ind_*_ult1`

---

### Продуктовые признаки

Все признаки вида:

```text
ind_*_ult1
```

— бинарные и показывают наличие продукта у клиента.

## Этап 1. Исследование данных (EDA)

`notebooks/01_eda.ipynb`

### Что было сделано

* анализ структуры данных
* оптимизация памяти
* анализ пропусков
* проверка дубликатов
* анализ категориальных признаков
* анализ активности клиентов
* анализ количества продуктов
* анализ популярности продуктов
* формирование таргета

---

### Ключевые выводы

#### Данные

* ~13.6 млн строк
* данные корректно структурированы
* дубликаты отсутствуют

#### Пропуски

* `conyuemp`, `ult_fec_cli_1t` → удаляются
* `renta` → важный признак, требует обработки

#### Клиенты

* большинство клиентов имеют длинную историю
* присутствует cold-start сегмент

#### Продукты

* большинство клиентов имеют 1–2 продукта
* продукты сильно несбалансированы
* наблюдается long-tail

#### Таргет

* крайне разреженный
* большинство значений = 0

---

### Итог EDA

Данные пригодны для построения рекомендательной системы, но требуют:

* обработки пропусков
* учета временной структуры
* применения ranking-подхода

## Этап 2. Инфраструктура (MLflow)

Для управления экспериментами используется MLflow.

---

### Конфигурация

* backend: SQLite
* artifact store: локальная директория
* порт: 5000

---

### Запуск

```bash
./scripts/run_mlflow.sh
```

---

### Доступ

Через SSH-туннель:

```bash
ssh -i <ssh_key> -L 5000:localhost:5000 <user>@<vm_ip>
```

После этого:

```text
http://127.0.0.1:5000
```

---

### Использование

```python
import mlflow

mlflow.set_tracking_uri("http://127.0.0.1:5000")
mlflow.set_experiment("bank_product_recommendation")
```

## Этап 3. Трансляция задачи

### Бизнес-задача

Рекомендовать клиенту продукты, которые он с высокой вероятностью оформит.

---

### ML-задача

Предсказать:

> новые продукты клиента в следующем месяце

---

### Таргет

```text
target = products_t - products_{t-1}
```

---

### Особенности

* разреженность
* дисбаланс
* multi-label

---

### Метрики

Используются ranking-метрики:

* Precision@K
* Recall@K
* MAP@K (основная)

---

### Стратегия

1. Формирование таргета
2. Бейзлайн (Top Popular)
3. Обучение модели (LightGBM)
4. Оценка по ranking-метрикам
5. Логирование в MLflow
